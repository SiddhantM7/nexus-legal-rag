System Architecture & Report

Detailed design specifications of the Nexus Legal Retrieval-Augmented Generation (RAG) System.

1. What is RAG?

RAG stands for Retrieval-Augmented Generation. Normally, an AI model answers questions entirely from its pre-trained memory, which often leads to hallucinations (guessing fake sections). RAG fixes this by first "Retrieving" exact legal text documents based on the user's query from our local FAISS database, and then "Augmenting" the prompt sent to the AI so the AI can "Generate" a precise answer based ONLY on those retrieved documents.

2. Why are embeddings used?

Computers do not understand raw words; they understand mathematical patterns. An embedding is a list of numbers (a dense vector) that represents the conceptual "meaning" of a chunk of legal text. Words with similar meanings will have vectors that are closer to each other in 768-dimensional mathematical space. We use embeddings to match the user's query meaning with the document's meaning, completely bypassing the limitations of exact keyword matching.

3. Document Chunking Strategy

Chunking is the process of breaking massive legal PDFs into smaller, bite-sized pieces so the vector database can process them. We opted for "Section-wise chunking" (using Regular Expressions to find "Section X" or "Article Y"). This ensures that legal contexts are perfectly preserved. If we were to cut text every 500 words blindly, a single law's definition could be split down the middle, causing the AI context to be destroyed.

4. FAISS Vector Database & Similarity Search

FAISS (Facebook AI Similarity Search) is an offline library providing rapid storage and lookup for dense vectors. When the user types a query, it is converted into a vector. FAISS performs an exact nearest-neighbor search (using L2 Euclidean distance) between the query vector and every vector in the index. The top 5 closest text chunks are returned and forwarded to the LLM backend.

5. Model Fine-Tuning & Local LLMs

The system utilizes nomic-embed-text-v1.5 for embeddings and mistral running purely locally via Ollama for generation. During development, a fine-tuning script is heavily integrated to auto-generate synthetic (Question, Context) pairs based on the PDFs, enforcing domain adaptation using MultipleNegativesRankingLoss so that the model deeply understands Indian Legal syntactic nuances.
